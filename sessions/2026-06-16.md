## 🔧 Vấn đề & Giải pháp

### 1. Naming Restrictions trong create-next-app
- **Triệu chứng:** Lệnh khởi tạo Next.js trực tiếp tại thư mục gốc báo lỗi do tên folder `Mayintuxa` có chữ viết hoa.
- **Nguyên nhân:** npm không cho phép tên package có chữ in hoa.
- **Giải pháp:** Khởi tạo tại folder tạm `app-temp`, sau đó di chuyển toàn bộ tệp (gồm cả tệp ẩn) ra thư mục gốc rồi xóa folder tạm.

### 2. Lỗi biên dịch dynamic params trên Next.js 15/16
- **Triệu chứng:** Khi chạy `npm run build`, API route `/api/admin/users/[id]` lỗi kiểu dữ liệu do `params.id` không được chấp nhận trực tiếp.
- **Nguyên nhân:** Từ Next.js 15+, thuộc tính `params` trong các route động là một `Promise` thay vì object thông thường.
- **Giải pháp:** Cập nhật tham số nhận vào là `Promise<{ id: string }>` và sử dụng `const { id } = await params`.

### 3. Lỗi require-time của thư viện ipp khi Next.js đóng gói (Turbopack)
- **Triệu chứng:** Next.js build crash tại bước thu thập dữ liệu do lỗi `Cannot read properties of undefined (reading 'cover-back')`.
- **Nguyên nhân:** Thư viện `ipp` chứa cấu trúc tham chiếu vòng tròn tại thời điểm require, bị lỗi khi đóng gói bằng Webpack/Turbopack.
- **Giải pháp:** Đưa `ipp` vào cấu hình `serverExternalPackages` trong `next.config.ts` để Next.js giữ nguyên require thô ở runtime, và cấu hình API route là `force-dynamic`.
