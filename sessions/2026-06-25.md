- **Fix `printer-state` enum:** Phát hiện `xref()` tạo object `{ 'idle': 3 }` (string→int), không phải `{ 3: 'idle' }`. Đổi `printer-state: 3` thành `printer-state: 'idle'`.

## 🔧 Vấn đề & Giải pháp

### 1. `Unknown attribute: device-uri` — ipp library không biết CUPS attributes
**Triệu chứng:** Đăng ký máy in báo lỗi `Unknown attribute: device-uri`.
**Nguyên nhân:** Thư viện `ipp` chỉ serialize RFC-2911 attributes. `device-uri`, `ppd-name` là CUPS extensions. Monkey-patch không hiệu quả vì Turbopack bundle module độc lập.
**Giải pháp:** Viết lại hoàn toàn IPP binary construction thủ công theo RFC 2910 wire format.
**File liên quan:** `src/lib/cups.ts`

### 2. `printer-state: 3` serialized sai
**Triệu chứng:** Enum `printer-state` có thể gây lỗi khi gửi số nguyên.
**Nguyên nhân:** `xref()` tạo `{ 'idle': 3 }` — lookup theo string key. Khi pass integer `3`, lookup trả về `undefined`.
**Giải pháp:** Dùng string `'idle'` thay vì số `3`. (Sau đó không còn liên quan vì đã viết binary thủ công.)
**File liên quan:** `src/lib/cups.ts`

### 3. HTTP 401 từ CUPS server
**Triệu chứng:** Sau khi fix IPP binary, CUPS trả về 401 với tất cả users thử (`print`, `admin`, `root`).
**Nguyên nhân:** CUPS mặc định bảo vệ `/admin/` và policy `CUPS-Add-Modify-Printer` bằng PAM authentication.
**Giải pháp:** Tạo `cups/config/cupsd.conf` tùy chỉnh — xóa `AuthType`/`Require user` khỏi các limit admin. Security layer do Next.js cung cấp.
**File liên quan:** `cups/config/cupsd.conf`

## 💡 Ghi chú / Insights
- IPP wire format (RFC 2910) khá đơn giản để implement thủ công: header 8 bytes + group tags + attribute TLV records. Không cần thư viện phức tạp cho việc ghi.
- Next.js Turbopack không share Node.js module cache giữa các phần của bundle — monkey-patch module cache không hoạt động như trong CommonJS thông thường.
- PWA với Next.js 16 App Router: chỉ cần `manifest.ts` + `sw.js` trong public + 1 component `useEffect` là đủ để install được app. Không cần plugin bên ngoài.
